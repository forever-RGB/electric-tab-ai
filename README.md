# Electric Tab AI - 电吉他自动扒谱网页

Electric Tab AI 是一个运行在浏览器中的 AI + 音频分析课程项目。用户上传音频或 MP4 文件后，网页会提取音频信号、检测主旋律音高，并自动映射到标准调弦电吉他的六线 TAB 谱。

## 功能说明

1. **音频/MP4 上传与预览**：支持拖拽或点击上传，浏览器可直接播放原始文件。
2. **浏览器端音高检测**：使用 Web Audio API 解码音频，通过自相关算法估计单音旋律频率。
3. **电吉他 TAB 自动生成**：将识别出的 MIDI 音高映射到 E A D G B e 标准调弦和指定最高把位。
4. **结果检查与导出**：展示音符明细、置信度、音域统计，并支持复制 TAB、下载 TXT 和 JSON。

## 运行环境

推荐浏览器：最新版 Chrome、Edge 或 Firefox。
不需要安装 Python、Node.js 或数据库。
如果直接双击 `index.html` 后浏览器限制音频解码，可以在项目目录运行一个本地静态服务器：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 使用方法

1. 打开 `index.html`。
2. 上传主旋律较清晰的音频文件，或带有音频轨道的 MP4 文件。
3. 根据音频特点调整“灵敏度”“最短音符”和“最高把位”。
4. 点击“开始扒谱”。
5. 在结果区复制或下载生成的 TAB 谱。

## 技术原理

项目使用 Web Audio API 的 `decodeAudioData` 将音频解码为 PCM 采样数据，再把多声道混合为单声道。程序按固定窗口切分音频，对每一帧做自相关计算，寻找最可能的周期，从而估计当前帧的基频。识别到的频率会转换为 MIDI 音高，再合并相邻且音高接近的帧，形成较稳定的音符片段。最后，程序根据标准调弦中每根弦的空弦 MIDI 值，为每个音符选择一个可弹奏的弦和品位，生成六线谱。

## 项目限制

本项目更适合单音旋律、独奏、哼唱旋律或背景较干净的电吉他片段。
如果音频包含复杂和弦、强鼓点、人声混响或多个乐器同时演奏，结果可能需要人工修正。
MP4 是否可解码取决于浏览器支持的编码格式。

## 学术诚信说明

本项目代码为课程作业场景下的原创实现，使用 AI 辅助进行结构设计、代码编写和文档整理。项目曾参考开源项目 [azuwis/pianotrans](https://github.com/azuwis/pianotrans/) 的产品思路：选择音频/视频文件并输出转写结果。但本项目没有复制其代码，技术路线也不同：pianotrans 主要面向钢琴转 MIDI 和桌面/命令行打包，本项目面向电吉他 TAB，并使用浏览器端 JavaScript 音高检测实现。核心算法、页面交互和项目文档均已在报告中说明，提交者应理解并能解释主要实现过程。
