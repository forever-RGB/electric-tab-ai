# Electric Tab AI 电吉他自动扒谱网页技术报告

## 一、项目背景

学习电吉他时，很多人听到喜欢的旋律后想把它扒成 TAB 谱，但人工扒谱需要乐理、听音能力和大量练习。专业扒谱软件通常安装复杂，有些还需要付费插件。为此，我制作了 Electric Tab AI：一个可以在网页中上传音频或 MP4 文件，并自动生成电吉他六线 TAB 初稿的 AI + 创意应用。

本项目把音频信号处理、音高识别、乐理映射和网页交互结合起来。用户不需要安装后端环境，只要打开网页，就可以完成“上传文件、分析旋律、查看 TAB、导出结果”的完整流程。调研时我参考了开源项目 azuwis/pianotrans 的产品思路，即选择音频/视频文件并输出转写结果；但本项目没有复制其代码，技术路线也不同：pianotrans 更偏向钢琴转 MIDI，本项目使用浏览器端 JavaScript 实现，输出目标是电吉他 TAB。

## 二、需求分析

根据课程要求，项目需要代码清晰、核心功能完整，并提供 README 和技术报告。本项目实现了四个基础功能。第一，音频/MP4 上传与预览，用户可以点击或拖拽文件，并用页面播放器试听原文件。第二，浏览器端音高检测，程序使用 Web Audio API 解码音频，再通过自相关算法估计单音旋律的基频。第三，电吉他 TAB 自动生成，把识别出的音高转换为 MIDI，并映射到标准调弦 E A D G B e 的弦和品位。第四，结果展示与导出，页面会显示音符数量、时长、音域、平均置信度、音符明细表，并支持复制 TAB、下载 TXT 和 JSON。

为了提升可用性，页面还提供“灵敏度”“最短音符”“最高把位”三个参数。不同音频的音量、噪声和演奏速度不同，用户可以通过这些设置调整检测效果。

## 三、关键代码实现说明

项目核心文件包括 `index.html`、`styles.css` 和 `app.js`。`index.html` 负责页面结构，包含上传区、播放器、参数控件、统计卡片、TAB 输出区和音符表格。`styles.css` 负责响应式布局和视觉样式。主要算法写在 `app.js` 中。

上传文件后，`loadFile` 会读取文件的 `ArrayBuffer`，创建 `AudioContext`，并调用 `decodeAudioData` 解码为 PCM 采样数据。由于音频可能有多个声道，`mixToMono` 会把所有声道平均混合成单声道，方便后续分析。

音高检测由 `detectPitchFrames` 和 `autoCorrelate` 完成。程序把音频切成短帧，先用均方根音量过滤安静片段，再计算自相关。自相关的思想是：如果信号有稳定周期，那么它与延迟一个周期后的自己会非常相似。程序找到相关性最高的延迟量，用 `sampleRate / lag` 得到频率，并转换成 MIDI 音高。随后 `mergeFramesIntoNotes` 会把相邻且音高接近的帧合并成更稳定的音符，减少抖动。

TAB 生成由 `chooseGuitarPosition` 和 `buildTab` 完成。标准调弦中六根弦的空弦 MIDI 值分别为 40、45、50、55、59、64。一个音的 MIDI 值减去某根弦空弦值，就得到该弦上的品位。如果品位在允许范围内，程序会选择更接近中低把位的位置，使结果更适合实际弹奏。最后按时间把音符放入六线谱文本中。

## 四、测试结果

本项目完成后进行了基础测试。页面可以通过本地静态服务器正常打开，上传区、播放器、按钮和结果区域显示正常。使用浏览器支持的音频文件时，系统可以解码并显示时长。点击“开始扒谱”后，程序能生成 TAB 文本和音符明细；当音频没有稳定旋律时，也会给出提示。复制、下载 TXT、下载 JSON 功能可用。缩小窗口后，页面会变成单列布局，主要内容不会重叠。

项目目前更适合单音旋律、哼唱旋律或背景较干净的电吉他片段。如果输入完整乐队混音、复杂和弦或强鼓点音频，结果可能需要人工修正，这是简化音高检测算法的限制。

### 五、出现问题及解决方案

#### 1. 音频解码兼容性问题

**问题：**

不同浏览器以及不同格式的 MP4 文件在解析时存在兼容性差异，部分文件上传后可能无法正常提取音频数据，导致后续分析失败。

**解决方案：**

使用 Web Audio API 的 `decodeAudioData()` 统一完成音频解码，并增加异常捕获机制。当文件格式不受支持或解码失败时，及时向用户显示提示信息，避免程序异常终止。

#### 2. 音高检测结果不稳定

**问题：**

实际音频中存在噪声、泛音和环境干扰，自相关算法得到的频率结果会出现抖动，导致同一个音符被识别为多个不同音高。

**解决方案：**

在音高检测前增加音量阈值过滤，去除静音和低能量片段；同时将相邻且音高接近的检测结果进行合并，生成更加稳定的音符序列，提高识别准确率。

#### 3. MIDI 到吉他指板映射困难

**问题：**

同一个 MIDI 音高在电吉他上往往对应多个可弹奏位置，直接映射容易产生跨度较大的指法，不符合实际演奏习惯。

**解决方案：**

设计吉他指板位置选择算法，在满足音高正确的前提下优先选择中低把位，并限制最高把位范围，使生成的 TAB 谱更加合理、易于演奏。

#### 4. 长音频分析效率问题

**问题：**

较长音频文件包含大量采样数据，分析过程耗时明显增加，用户等待时间较长。

**解决方案：**

采用分帧处理方式进行音高检测，并增加最短音符参数控制，减少无效计算量。同时在界面增加分析状态提示，提升用户体验。

#### 5. Python 运行时中文符号报错

**问题：**

在编写 Notebook 和项目文档时，误将中文全角符号直接写入 Python 代码，出现如下错误：

```
SyntaxError: invalid character '（' (U+FF08)
```

**解决方案：**

将代码中的中文全角符号统一替换为英文半角符号，例如：

```
（ → (
） → )
： → :
， → ,
```

对于说明性内容，改用 Markdown 文本或 Python 注释形式书写，避免被解释器误认为可执行代码。

## 六、总结

Electric Tab AI 实现了从音频到电吉他 TAB 初稿的自动化流程，满足课程中“AI + 创意项目”的方向。通过本项目，我理解了音频解码、PCM 采样、自相关音高检测、MIDI 转换和吉他指板映射等知识。后续可以继续加入波形显示、节拍检测、手动编辑和更高级的神经网络音高模型，提高复杂音乐中的准确率。
